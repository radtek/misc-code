This is John's Very Lame VBScript page to show calling the TraceSrv DCOM Service.

Special Note: If you configure TraceSrv to run through DCOM on another machine, Internet Explorer will properly create the TraceSrv object and you can call through it just fine. However, when you exit Internet Explorer 3.02 (4.70.1300), Internet Explorer will crash with a bad memory read at EIP 0x77A83450 on Intel machines at and address FP 0x776E6020 on ALPHA machines. The crash is in SECURITY.DLL. I have not tried this on Internet Explorer 4.0 betas.
